Features

Everything you need to
own your API performance

Built for developers who know autocannon and want to stop copy-pasting flags into a terminal.

📁

Project Management

Organise tests into projects. Each project holds multiple named configurations, each with its own result history. Switch between them in one click.

⚙️

Full Autocannon Config

Every option autocannon exposes — connections, pipelining, rate limiting, multi-request scenarios, bailout, socket paths — available through a clean tabbed form. No CLI flags.

📈

Real-Time Charts

Eight live charts update every second: req/s, throughput, latency percentiles (p50/p75/p90/p99), status codes, errors, cumulative requests, and avg response size.

🎯

Latency Percentiles Live

p50, p75, p90, and p99 computed per second from actual response times — not approximations. Watch how your tail latency behaves under sustained load.

📦

Result History & Export

Every completed test is stored automatically. Browse past runs, compare metrics, and export any result to JSON or CSV with a native save dialog.

🖥️

Console Logging

Every tick printed to the terminal where the app launched: req/s, throughput, errors, p50, p99. Full summary on completion — great for CI output.

Visualisations

How it works

From zero to benchmark
in under a minute

01

Create a project

Group related services together. Give the project a name and optional description — it's just an organisational folder.

02

Configure your test

Set the URL, method, connections, duration, headers, body, and any advanced option through the tabbed form. Save it with a name.

03

Hit Run

Charts fill in live. Watch latency percentiles, req/s, and status codes update every second. Stop early anytime.

04

Analyse & export

Browse the full result: latency histogram, request totals, error breakdown. Export to JSON or CSV. History saved automatically.
